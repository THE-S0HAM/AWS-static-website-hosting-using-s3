How to Host a Static Website Using Amazon S3

1. Create an AWS account

Sign up for an AWS account if you don't already have one. You'll need to provide payment information, but there's a free tier available.

2. Open the S3 console and create a bucket

Navigate to the S3 service in the AWS Management Console and click "Create bucket". Choose a globally unique bucket name and select your preferred region.

3. Uncheck "Block All Public Access"

For a public website, you'll need to allow public access to your bucket. Uncheck the "Block all public access" option during bucket creation.

4. Upload your index.html and other files

Upload all your website files including HTML, CSS, JavaScript, and any media files to your bucket.

5. Enable Static Website Hosting in the bucket properties

Go to the bucket properties, find the "Static website hosting" section, and enable it. Set your index document (usually index.html) and error document.

6. Add a public-read bucket policy to allow file access

Add a bucket policy that grants public read access to all objects in your bucket. This is necessary for visitors to view your website.

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::your-bucket-name/*"
    }
  ]
}

7. Copy the website endpoint URL to view your site

After enabling static website hosting, S3 provides an endpoint URL. Use this URL to access your website.

Best Practices

  • Use compressed images to reduce load time
  • Keep file names lowercase and consistent
  • Test the site locally before uploading
  • Avoid hardcoding sensitive information
  • Use meaningful tags for HTML (semantic markup)
