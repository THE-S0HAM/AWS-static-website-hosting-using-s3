Amazon S3 Technical Documentation

Key Features of Amazon S3

Storage Classes

Amazon S3 offers multiple storage classes designed for different use cases:

  • S3 Standard: For frequently accessed data
  • S3 Standard-IA: For infrequently accessed data
  • S3 Glacier: For long-term archival
  • S3 Express One Zone: High-performance, single-zone storage with single-digit millisecond latency

How Amazon S3 Works

Amazon S3 stores data as objects within buckets. An object consists of a file and metadata that describes the file. A bucket is a container for objects.

  • Objects: Files and their metadata stored in S3
  • Buckets: Containers that hold objects
  • Keys: Unique identifiers for objects within a bucket
  • Regions: Geographic locations where S3 stores your buckets

Data Consistency Model

Amazon S3 provides strong read-after-write consistency for PUT and DELETE requests in all AWS Regions. This means:

  • Immediate consistency for new object writes
  • Immediate consistency for object updates and deletes
  • Strong consistency for list operations

Security and Access Management

S3 provides multiple layers of security:

  • IAM Policies: Control access at the user and role level
  • Bucket Policies: Resource-based policies for bucket-level access
  • Access Control Lists (ACLs): Object-level access control
  • S3 Block Public Access: Prevents accidental public exposure

Accessing Amazon S3

Multiple ways to interact with S3:

  • AWS Management Console: Web-based interface
  • AWS CLI: Command-line interface
  • AWS SDKs: Programming language-specific libraries
  • REST API: Direct HTTP interface

Paying for Amazon S3

Amazon S3 charges you only for what you actually use, with no hidden fees and no overage charges. This pay-as-you-go model gives you a variable-cost service that can grow with your business.

  • No predetermined storage capacity requirements
  • No overage fees or service shutoffs
  • Cost advantages of AWS infrastructure
  • AWS Free Tier available for new customers

When you sign up for AWS, your account is automatically signed up for all services, but you're only charged for what you use.

PCI DSS Compliance

Amazon S3 supports the processing, storage, and transmission of credit card data and has been validated as compliant with Payment Card Industry (PCI) Data Security Standard (DSS).

  • Validated for PCI DSS Level 1 compliance
  • Suitable for merchant and service provider use
  • AWS PCI Compliance Package available upon request
